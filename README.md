# 字幕快速清除工具

使用 Python + OpenCV 的可视化小工具，可批量载入视频，逐个视频框选字幕区域并使用高斯模糊处理。支持拖动时间轴预览、逐个视频独立编辑，并在本地尝试保留音频（需要本机安装 `ffmpeg`）。

## 环境准备

1. 建议 Python 3.9+。
2. 安装依赖：
   ```bash
   pip install -r requirements.txt
   ```
3. 可选：安装 `ffmpeg` 以在导出时自动合并原始音轨。未安装 `ffmpeg` 时仍可导出视频，但会丢失音频。

## 运行

```bash
python main.py
```

## 使用步骤

1. 点击 “添加视频” 选择一个或多个视频文件。
2. 在右侧列表选择要编辑的视频。
3. 在 “遮罩设置” 中点击 “＋” 添加一个遮罩，然后在预览画面中拖拽框选要遮盖的区域。**拖动过程中即可实时看到遮罩效果**，松开鼠标即固定，所见即所得。
4. 每个视频可添加**多个遮罩**（如同时遮字幕与台标），用遮罩下拉框切换编辑对象；“－” 删除当前遮罩。
5. 每个遮罩有**独立的算法与强度**：选中某个遮罩后，调整 “算法 / 强度” 即作用于该遮罩。
6. **移动遮罩（跟随浮动水印）**：拖动时间轴到另一帧，重新框选该遮罩的位置，即自动生成关键帧；两帧之间的位置线性插值。可设多个关键帧。
   - “删除关键帧” 删除当前帧的关键帧；“转为静态” 把移动遮罩压缩回固定位置。
7. 如需导出时去除音轨，勾选 “去除音轨”；默认会尝试保留原音频（需要 `ffmpeg`）。
8. 默认会自动检测视频比例，若不是 9:16 会裁剪到 9:16（从中心裁剪，无黑边）；可取消勾选 “裁剪为 9:16” 以保持原比例。
9. 若需自定义导出目录，点击 “选择”；未选择时默认导出到源文件所在目录。
10. 勾选 “导出后从列表移除” 可在导出完成后自动从程序列表移除对应视频（不影响源文件）。
11. 未添加遮罩时也可直接导出（仅应用裁剪/音频/比例设置）。
12. 编码器可选 `自动检测`（默认，优先 NVENC / QSV / AMF，失败自动回退软件 x264）、`h264_nvenc`、`h264_qsv`、`h264_amf`、`libx264`、`copy`（需 ffmpeg 支持对应硬件编码）。
13. 点击 “开始处理” 导出列表中的视频。输出文件名称追加 `_blurred` 后缀。
14. 如果自动检测不到 ffmpeg，可在导出选项里点击 “浏览” 手动指定可执行文件。

## 已知说明

- 输出默认使用 OpenCV 写入 MP4；若本机未安装 `ffmpeg`，导出的视频将没有音频。
- 导出会通过 `ffprobe` 读取源视频码率，并以目标码率 + maxrate/bufsize 的方式尽量保持原有码率；无法读取时回退到质量优先（CRF/CQ）模式。
- 充分利用显卡：解码优先走 CUDA / D3D11VA / DXVA2 / VAAPI 硬件加速，模糊在可用时使用 OpenCV CUDA 滤镜，编码优先使用 NVENC / QSV / AMF 硬件编码器。
- **融合式 GPU 管线**：当一个视频的遮罩都是「静态矩形模糊/马赛克」时，导出会用单条 ffmpeg 命令完成「硬件解码 → 区域滤镜 → NVENC 编码」，不再逐帧回传 Python、也没有进程间 rawvideo 拷贝，CPU 占用大幅下降；音轨直接拷贝不重编码。移动遮罩、inpaint、时域擦除、纯色/中值/毛玻璃仍走逐帧管线（该管线也会尽量使用硬件解码 + NVENC 编码）。日志会标明每个视频实际走了哪条管线、解码/编码是否在 GPU 上。
- 关于 `cv2.cuda`：官方 `opencv-python` 轮子不含 CUDA，其 GPU 滤镜路径不会生效；如需 OpenCV 侧 GPU 模糊，需要自行编译带 CUDA 的 OpenCV。融合式 GPU 管线不依赖它。
- 框选区域与模糊设置在同一视频内复用，可在时间轴任意帧预览模糊效果。
- GUI 会根据当前屏幕 DPI 自动缩放，适配高分屏。
- 右侧“日志”面板可查看处理进度与 ffmpeg 回退情况。（日志位于列表下方）
- 单窗口布局：左侧为预览与设置，右侧为列表与日志，分隔条可拖动调整比例。

## 遮罩 / 擦除方式

算法分两类：

- **覆盖式遮挡**（`高斯模糊`、`中值/均值模糊`、`双重模糊`、`像素化 / 强像素化 / 像素化+高斯`、`毛玻璃`、`纯色条`）：遮住整块选区，会一并覆盖选区内的真实画面。默认叠加**羽化软边**，让遮挡区与周围画面平滑过渡，不再是生硬的矩形边界。模糊核大小**随框选区域自适应**：字幕框越大核越大，保证 1080p 及以上的大号字幕也能被彻底抹平；融合 GPU 管线与逐帧路径的滤镜强度已逐一对齐（预览=导出）。
- **擦除式**（`字幕擦除(inpaint)`、`字幕擦除(时域)`）：只识别并处理字幕笔画，尽量保留字母之间与周围的真实背景，符合“遮住字幕但不破坏背景”的目标。
  - `字幕擦除(inpaint)`：逐帧检测高对比文字笔画后用 `cv2.inpaint` 就地修复；适合背景不过于复杂的画面。笔画过大过密或背景极复杂时会有涂抹感，此时可改用覆盖式遮挡。
  - `字幕擦除(时域)`：先采样多帧估计选区的“干净背景”（中值），再逐帧仅替换字幕像素；当字幕区背景基本静止或字幕半透明时效果更好，代价是导出前多一次采样、更耗内存。预览时该模式按空间 inpaint 效果显示。

> 说明：硬字幕烧录在画面上，笔画处的原始背景已不存在，擦除式为“重建”的近似结果，并非无损还原；背景越复杂，重建痕迹越明显。

## 代码结构

- `main.py`：图形界面与交互逻辑。
- `ui_widgets.py`：可复用 UI 小部件（宽度感知截断 `ellipsize_middle`、悬停提示 `Tooltip`、双向滚动容器 `XYScrollFrame`）。
- `theme.py`：Material Design 3 深色主题（参考 Google 设计规范）——表面层级、语义色、圆角与各类控件的配色组合，全部界面颜色集中于此。
- `app_config.py`：配置文件的读写与旧版本迁移（`AppConfig`）。
- `media_tools.py`：ffmpeg / ffprobe 探测、硬件解码/编码检测、源码率读取、编码参数生成。
- `exporter.py`：`VideoExporter` 导出管线（融合式 GPU 管线优先，回退逐帧：解码 → 遮罩 → 可选 9:16 裁剪 → 编码），含硬件→软件编码回退。
- `ffmpeg_pipeline.py`：融合式 GPU 管线的滤镜图/命令生成（静态矩形模糊/马赛克，硬件解码 + NVENC 编码单进程完成）。
- `masks.py`：遮罩数据模型（`Mask`），支持单视频多遮罩、每遮罩独立算法/强度、关键帧运动（线性插值）与序列化。
- `blur_core.py`：遮挡/擦除算法实现（模糊、马赛克、毛玻璃、纯色条、羽化软边、字幕检测 + inpaint、时域背景合成、多遮罩按帧应用 `apply_masks`），支持 OpenCV CUDA 加速。
- `perf_benchmark.py`：模糊算法的 CPU 性能基准脚本。
